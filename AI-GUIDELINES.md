# AI Usage Guidelines

This document outlines the rules and best practices for using AI tools (e.g., Vibe Coding) in the **go53** project.

---
## Allowed Use Cases

- **Tests**: AI tools may be used to generate, improve, or debug test code (unit tests, integration tests, etc.).
- **Documentation**: AI tools may be used to create or update documentation, such as README files, comments, or API documentation.

---
## !!Prohibited Use Cases!!

- **Core Project Code**: AI tools **must not** be used to generate or modify core project logic. All core code must be written manually by developers.

---
## Review and Iteration

- All code generated by AI tools (even for tests and documentation) **must** be manually reviewed before merging.
- Use AI tools as a **support**, not as a replacement for critical thinking.

---
## Rationale

- **Quality and Accountability**: Core code is the backbone of the project. Manual development ensures understanding, maintainability, and accountability.
- **Transparency**: Tests and documentation are easier to review and validate, minimizing risks.

---
## Best Practices

- Always verify AI-generated output for accuracy and relevance.
- Document any AI-assisted changes in commit messages or PR descriptions (e.g., "Generated test cases using Vibe Coding, manually reviewed").
- Avoid using AI for complex logic or architectural decisions.
